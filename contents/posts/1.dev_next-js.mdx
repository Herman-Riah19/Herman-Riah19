# Getting Started with Next.js Development

Next.js is a powerful **React framework** that enables developers to build fast, scalable, and SEO-friendly web applications.  
It simplifies modern development by combining **frontend and backend logic** in one unified codebase.

## ðŸ§­ Pages and Layouts

Next.js introduced the **App Router** (from version 13 onwards), where each folder inside `/app` becomes a route.

### Example of a Basic Page

```tsx
// app/about/page.tsx
export default function AboutPage() {
  return (
    <div className="p-8">
      <h1 className="text-3xl font-bold">About Us</h1>
      <p className="text-muted-foreground mt-2">
        We are a team of passionate developers building with Next.js.
      </p>
    </div>
  );
}
```

This automatically maps to `/about`.

## ðŸª„ Server Actions (Next.js 14+)

**Server Actions** allow you to perform backend logic directly in your components without a traditional API endpoint.

```tsx
"use server";

import { revalidatePath } from "next/cache";
import { prisma } from "@/lib/db";

export async function addTodo(formData: FormData) {
  const title = formData.get("title") as string;
  await prisma.todo.create({ data: { title } });
  revalidatePath("/todos");
}
```

You can then call this function directly from a React component:

```tsx
<form action={addTodo}>
  <input name="title" placeholder="New todo..." />
  <button type="submit">Add</button>
</form>
```

## ðŸ§© Middleware

The `middleware.ts` file runs **before a request is processed** â€” useful for:

* Authentication
* Redirects
* Logging
* Localization

Example:

```ts
// middleware.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function middleware(request: NextRequest) {
  const token = request.cookies.get("token")?.value;

  if (!token && request.nextUrl.pathname.startsWith("/dashboard")) {
    return NextResponse.redirect(new URL("/login", request.url));
  }

  return NextResponse.next();
}
```

You can also limit where it runs:

```ts
export const config = {
  matcher: ["/dashboard/:path*", "/settings/:path*"],
};
```
